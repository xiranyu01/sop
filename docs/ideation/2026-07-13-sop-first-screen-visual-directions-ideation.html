Ideation

SOP 首屏视觉方向

在完全保留当前信息架构、业务逻辑和交互路径的前提下，用成熟产品设计系统的表面语言改善首屏：收敛颜色、减少装饰、增强数据层级。

日期：2026-07-13 主题：sop-first-screen-visual-directions 焦点：低改动、简洁现代、1–3 个主题色 模式：repo-grounded
6最终候选
3视觉轴
CSS-first推荐落地方式

Evidence

Grounding Context

Codebase Context：当前首屏是固定 240px 侧栏、顶部标题/刷新、单一搜索工具栏和九列数据表。样式集中于 src/App.css，业务集中于 src/App.tsx；没有 UI 框架和 token 层，因此适合 CSS-first 换肤。

现有 navy、teal、yellow、blue 四处竞争注意力；同时 panel 已有边框、分隔线和轻阴影，层级表达略重复。低风险改造应保留全部结构，只重做颜色角色、边框、圆角、阴影、字体层级和表格密度。

Surface map

Topic Axes

  • Navigation / chrome：品牌、侧栏、激活态与计数。
  • Content surfaces / elevation：画布、panel、工具栏、边界与阴影。
  • Table density / hierarchy：表头、行高、数字、状态与扫描效率。

Candidates

1.

Primer Precision

推荐起点

冷白画布、浅灰侧栏、6px 几何、发丝边框与单一蓝色。它最接近现有企业后台骨架，但能一次解决多强调色和卡片模板感。

Axis
Navigation / chrome
Basis
direct: 当前导航已有独立 active/count seam；external: GitHub Primer 是直接适用于产品工具界面的 foundations/patterns。
Rationale
风险最低，且共享 token 可覆盖侧栏、按钮、链接、focus 和表格 selected 状态。
Downsides
浅色侧栏会弱化当前深海军蓝的品牌边界，整体也相对保守。
Confidence
95%
Complexity
Low
2.

Atlassian Structured

复杂工作流

保留深蓝侧栏，以白色内容面、清晰容器边界和规则间距建立强层级。它比 Primer 更接近当前使用习惯，也更适合后续详情页的复杂表单。

Axis
Content surfaces / elevation
Basis
direct: 当前 24px workspace 与 8px 圆角已有规则骨架；external: Atlassian Foundations 强调 token 化间距和可预测层级。
Rationale
深侧栏连续性好，表格与详情页都容易沿同一规则扩展。
Downsides
视觉上最“企业工具”，现代感不如 Linear/Vercel 强。
Confidence
91%
Complexity
Low
3.

Carbon Compact

数据效率

冷灰分层、0–4px 圆角、42px 数据行和更明确的中性表头，把首屏塑造成精确的数据工作台。九列结构、横向滚动和操作位置保持不变。

Axis
Table density / hierarchy
Basis
direct: DataTable 的 header/cell/badge/hover 均有独立 CSS seam；external: Carbon Data Table 提供直接的企业数据表先例。
Rationale
表格是首屏核心，高密度规则还会复用到客户、物料、场景和机器型号页面。
Downsides
方正与高密度可能显得偏硬，需要重点检查中文字号和点击舒适度。
Confidence
90%
Complexity
Low
4.

Linear Soft

现代精致

深石墨侧栏、克制紫蓝、hairline border 和极轻浮层，在不更改布局的情况下提供最明显的“新产品感”。

Axis
Navigation / chrome
Basis
direct: 全局颜色、圆角与阴影均集中在 CSS；external: 社区 DESIGN.md 分析，已明确标注为 inspired 而非官方规范。
Rationale
视觉差异足够大，适合作为“明显现代化”的对照组。
Downsides
紫色与现有 coScene 青绿/蓝习惯距离更远，品牌选择需要额外确认。
Confidence
86%
Complexity
Low
5.

Vercel Monochrome

最简极限

黑白灰为主、零阴影、细网格和强排版层级；状态色只保留语义用途。它测试产品是否可以完全依靠结构而非品牌色数量成立。

Axis
Content surfaces / elevation
Basis
reasoned: 当前单一主 panel 已有足够分隔线，不依赖阴影也能建立层级；external: 社区 Vercel-inspired 分析，非官方规范。
Rationale
六个方案中最简、辨识度最高，能清楚显示团队对“极简”的接受边界。
Downsides
会大幅弱化现有品牌色习惯，最终全局落地需要更细致地处理状态和危险操作。
Confidence
82%
Complexity
Medium
6.

Warm Editorial

阅读友好

暖白暖灰、深墨文字、低对比边框和克制青绿，以编辑台账的排版与线条组织版本、日期和需求信息。

Axis
Table density / hierarchy
Basis
direct: 首屏的标题—索引—条目结构与台账相似；external: 社区 Notion-inspired 分析仅作为暖表面参考，不宣称官方复刻。
Rationale
它提供唯一的暖色阅读路线，适合长期归档和逐条审阅。
Downsides
与当前冷蓝习惯距离较远，且如果文字层级控制不好会退化为“普通暖色后台”。
Confidence
75%
Complexity
Low
